# Mobile — LoanPackagePreviewScreen

> Phase: 6C | Owner: ui-ux-agent | Date: 2026-04-25

---

## Purpose

Final review of the PDF package that will be sent to the partner bank. The user paginates through the merged document, verifies content, and submits. Submission requires biometric re-authentication and writes the immutable submission record.

## User Goal

"Show me the exact PDF the bank will receive. I want to confirm nothing is wrong before pressing submit."

---

## Entry guard

- Screen requires biometric re-auth on mount (LocalAuthentication). Failed/cancelled bio → return to LoanApplicationScreen with toast "Authentication required to view package".
- This protects PII (GSTIN, PAN, Aadhaar last 4, bank statement) from device-shoulder-surfing.

## Components used

- `PackageMetaStrip` — small top strip; pages, size, generated time, package ID.
- `PdfViewerMobile` (new — see component-library addendum) — wraps `react-native-pdf` with watermark visibility check + accessibility text layer.
- `PageIndicator` — floating chip at bottom-right, 32pt height.
- `AccordionSection` (Phase 6D) for "What's inside".
- `DisclaimerCard` — uses `Callout` info variant. Mandatory legal copy.
- `StickyFooter` with Submit CTA.

## Watermark verification

- Each rendered page includes the diagonal watermark: `"Generated by SnapAccount | {orgName} | {date} | Package ID: {id} | Not a CA certification"`.
- The viewer must not allow watermark removal. Visibility test runs on first page render — if watermark text is not detectable (page text-layer scan), show error banner "Package failed integrity check. Please regenerate."

## Submit flow

1. Tap `Submit to {Bank}` → confirm modal:
   - Title "Send package to {Bank}?"
   - Body "Once sent, you cannot edit your application. The bank may take 3–7 business days to respond."
   - Buttons: `Cancel` / `Send package`.
2. On confirm → biometric re-auth (second time, since the first was just to view).
3. On bio success → POST `/loans/{appId}/submit`. Show full-screen pending state (3–10s spinner with copy "Securing your submission…").
4. On success → navigate to LoanStatusScreen with status `SUBMITTED` (slide-up animation). Push notification arrives once bank acknowledges (`UNDER_REVIEW`).
5. On error → toast + retry; remain on screen.

## States

- **Generating** — first mount may show "Building package…" full-screen progress (poll backend; should resolve within 30s per scope §Exit#3). Show subtle progress bar + tip "Compiling 12 months of GSTR-3B…".
- **Ready** — full UI as above.
- **Generation failed** — full-screen error with Retry → re-runs ReportService PDF generation.
- **Stale** — if user edits docs after package generated, banner "Documents changed. Regenerate package." with `Regenerate` button at top.
- **Submitting** — modal-blocking spinner.
- **Already submitted** — Submit CTA replaced with "Already submitted on {date}" + link to LoanStatusScreen.

## Indian-format

- Date+time: DD MMM YYYY HH:mm IST.
- Size: MB with 1 decimal.
- Package ID: `PKG-YYYY-MM-{n}` (server-issued).

## Disclaimer copy (required)

> "Prepared by SnapAccount from user-provided data. Not a CA certification. Final lending decision rests with the partner bank."

This text MUST appear (a) as DisclaimerCard above footer, (b) on every PDF page footer, and (c) in the email body when EmailBankAdapter sends the package. (Source: phase-6C-scope risk #6 + plan Module F legal review.)

## Accessibility

- PDF viewer exposes per-page text via accessibility text layer (uses PDF's text content, not OCR).
- PageIndicator announces page changes (`liveRegion='polite'`).
- DisclaimerCard read in full when focused; do not truncate.
- Submit CTA includes bank name in label.
- Touch targets 44×44pt; CTAs full-width inside footer.
- Reduced-motion: skip slide-up to LoanStatusScreen.
