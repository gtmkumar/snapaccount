## SEC-056 Status — 2026-06-11

**Verdict: WIRED (fully implemented)**

### Background

SEC-056 was filed in Phase 6F as LOW severity: the admin `settingsApi.ts` frontend made calls to five PATCH endpoints (`/auth/feature-flags`, `/auth/config/ai`, `/auth/org/settings`, `/auth/config/language`, `/auth/config/whatsapp`) that did not exist in any backend service — "ghost endpoints" that created a future permission-bypass risk when stubs would eventually be added.

The finding was deferred as P6-HANDOFF-36 and the gap-analysis-2026-06-10 listed it as OPEN-DEFERRED. The gap-analysis-2026-06-11-delta.md noted "SEC-056 ghost-route wiring PARTIAL — backing tables exist; admin endpoints deferred".

### Current State (2026-06-11)

Code inspection of the live backend on branch `2026-06-10-s5t4` (commit `75c0e69`) reveals **all SEC-056 routes are now implemented**:

**`backend/Services/AuthService/AuthService.Api/Endpoints/Settings.cs`** (created in Wave 2):

| Frontend call | Backend route | RBAC guard |
|---|---|---|
| `GET /auth/org/settings` | `MapGet("/org/settings", GetOrgSettings)` | `[RequiresPermission("org.settings.read")]` on `GetOrgSettingsQuery` |
| `PATCH /auth/org/settings` | `MapMethods("/org/settings", ["PATCH"])` | `[RequiresPermission("org.settings.update")]` on `UpdateOrgSettingsCommand` |
| `GET /auth/feature-flags` | `MapGet("/feature-flags", GetFeatureFlags)` | `[RequiresPermission("platform.feature-flags.read")]` on `GetFeatureFlagsQuery` |
| `PATCH /auth/feature-flags/{flag}` | `MapMethods("/feature-flags/{flag}", ["PATCH"])` | `[RequiresPermission("platform.feature-flags.write")]` on `SetFeatureFlagCommand` |
| `GET /auth/config/language` | `MapGet("/config/language", GetLanguageConfig)` | `[RequiresPermission("platform.config.read")]` on `GetPlatformConfigQuery` |
| `PATCH /auth/config/language` | `MapMethods("/config/language", ["PATCH"])` | `[RequiresPermission("platform.config.write")]` on `UpdatePlatformConfigCommand` |
| `GET /auth/config/whatsapp` | `MapGet("/config/whatsapp", GetWhatsAppConfig)` | `[RequiresPermission("platform.config.read")]` on `GetPlatformConfigQuery` |
| `PATCH /auth/config/whatsapp` | `MapMethods("/config/whatsapp", ["PATCH"])` | `[RequiresPermission("platform.config.write")]` on `UpdatePlatformConfigCommand` |

**`backend/Services/AuthService/AuthService.Api/Endpoints/AiConfigEndpoints.cs`** (also new in Wave 2):

| Frontend call | Backend route | RBAC guard |
|---|---|---|
| `GET /auth/config/ai` | `MapGet("/config/ai")` | `[RequiresPermission("platform.ai.manage")]` |
| `PATCH /auth/config/ai` | `MapPatch("/config/ai")` | `[RequiresPermission("platform.ai.manage")]` |
| `GET /auth/config/ai/usage` | `MapGet("/config/ai/usage")` | `[RequiresPermission("platform.ai.manage")]` |
| `POST /auth/config/ai/test` | `MapPost("/config/ai/test")` | `[RequiresPermission("platform.ai.manage")]` |

### Evidence

- `backend/Services/AuthService/AuthService.Api/Endpoints/Settings.cs` — 8 routes implemented, file comment explicitly references "SEC-056: implements the previously ghost routes that the frontend settings pages were calling without a backend implementation."
- `backend/Services/AuthService/AuthService.Api/Endpoints/AiConfigEndpoints.cs` — 7 additional AI config routes.
- All handlers carry `[RequiresPermission(...)]` via MediatR `PermissionBehavior` — the specific risk noted in SEC-056 ("when stubs are added, `[RequiresPermission("admin.settings.*")]` gate required") has been addressed.
- Permissions are catalogued in `backend/Services/AuthService/AuthService.Domain/Permissions.cs`: `OrgSettingsRead`, `OrgSettingsUpdate`, `PlatformFeatureFlagsRead`, `PlatformFeatureFlagsWrite`, `PlatformConfigRead`, `PlatformConfigWrite`, `PlatformAiManage`.
