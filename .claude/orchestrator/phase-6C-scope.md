## Why this is P0

Loan Hub is a signature feature: "Apply for a business loan with one tap — we'll package your 12 months of GSTR-3B + P&L + Balance Sheet + bank summary + KYC into a single watermarked PDF and route to partner banks." Today 0 handlers, all 6 endpoints 501, no PDF generator, no bank adapters.

## Scope

### db-engineer (additive)

- `loan.loan_products` — partner bank's products catalog (bank_id, product_name, min/max amount, interest_rate_range, tenure_months, eligibility_criteria_jsonb).
- `loan.applications` — id, org_id, user_id, loan_product_id, requested_amount, tenure_months, purpose, status enum (DRAFT/SUBMITTED/UNDER_REVIEW/DOCS_REQUESTED/APPROVED/REJECTED/DISBURSED/CLOSED), submitted_at, bank_reference_no, disbursed_at, disbursed_amount.
- `loan.application_documents` — app_id, document_type (PAN/AADHAAR/GSTR3B/PL/BS/BANK_STMT/ITR/TRADE_LICENSE), document_id (FK documents.id), status.
- `loan.consents` — app_id, consent_type (CREDIT_BUREAU/DATA_SHARE_WITH_BANK/DISBURSEMENT_MANDATE), consent_text_version, signed_at, ip_address, user_agent, signature_hash.
- `loan.partner_banks` — bank_id, name, logo_url, adapter_type (EMAIL/REST/OAUTH), contact_email, api_config_encrypted.
- `loan.application_status_log` — full audit of status transitions.
- `loan.pdf_packages` — app_id, gcs_uri, pages_count, sha256_hash, generated_at, generated_by, watermark_text.
- Indexes + RLS (user sees their org's apps; CA sees assigned; bank API tokens masked).

### backend-agent

1. **LoanService full build:**
   - Domain: `LoanApplication`, `LoanProduct`, `Consent`, `PartnerBank`, `ApplicationStatus` state machine, `LoanPackage`.
   - Application: Commands (StartApplication, UpdateApplication, AttachDocument, RecordConsent, SubmitApplication, AssignToBank, ReceiveBankDecision, RecordDisbursement, CloseApplication) + Queries (GetApplication, ListApplications, CheckEligibility, GetPartnerBanks).
   - Consent service: DPDP-compliant, text version tracked, signature = HMAC(user_id + app_id + consent_text_version + timestamp, server_key).
   - Eligibility engine: reads AccountingService reports + GstService returns to compute eligibility score; engine rules configurable via `loan.eligibility_rules` table (versioned).
2. **ReportService PDF generation:**
   - Choose **QuestPDF** (MIT-licensed, .NET-native, server-friendly, no Chromium overhead).
   - Templates: TrialBalance.pdf, ProfitAndLoss.pdf, BalanceSheet.pdf, CashFlow.pdf, TaxLiability.pdf, LedgerByAccount.pdf, LoanPackage.pdf (composite).
   - Loan package: pull last 12 GSTR-3B + last FY P&L + last FY BS + last 6 months bank summary + KYC → merge → watermark: `"Generated by SnapAccount | {org_name} | {date} | Package ID: {id} | Not a CA certification"`.
   - Upload to GCS under `gs://snapaccount-{env}-docs/loan-packages/{app_id}/{package_id}.pdf`. Short-lived signed URL for user/bank download.
3. **Bank Adapter pattern:**
   - `IPartnerBankAdapter` interface: `SubmitApplicationAsync(LoanApplicationDto, Stream packagePdf) → BankSubmissionResult`.
   - Reference implementation 1: `EmailPartnerBankAdapter` — formats email, attaches PDF, sends via SendGrid to bank's contact email. Tracks message_id for audit.
   - Reference implementation 2: `RestPartnerBankAdapter` — generic REST POST with OAuth2 client-credentials; signed payload.
   - Registry: DI keyed on `partner_banks.adapter_type`.
4. **Disbursement webhook** receiver: `POST /loans/webhooks/{bankId}/disbursement` — HMAC verified, updates application status, emits `LoanDisbursedEvent` → Notification.
5. All 6 Loan endpoints mediator-wired; 0 501 responses; 0 TODO markers.
6. Tests ≥80%.

## Exit Criteria

1. User applies for loan on mobile end-to-end: catalog → eligibility → consent → application → package preview → submit.
2. Consent captured with signed HMAC; `loan.consents` row created per consent type.
3. PDF package generated within 30s of submit; ≤ 10 MB; watermark rendered on every page; readable on mobile PDF viewer.
4. Admin sees submitted app; assigns to partner bank; EmailBankAdapter sends PDF to bank email; audit logged.
5. Disbursement webhook (simulated) → application status DISBURSED → user gets push + SMS + "Loan Disbursed" celebration screen.
6. 0 501 responses from LoanService or ReportService.
7. Tests ≥80% LoanService + ReportService.
8. Zero new Critical/High security findings.

## Dependencies & Risks

- **CA certification language** (gap analysis risk #6) — legal disclaimer required on package footer. Copy: "Prepared by SnapAccount from user-provided data. Not a CA certification. Final lending decision rests with the partner bank."
- **Real partner bank contracts** — reference adapters suffice for MVP; commercial deals out of scope. Email fallback covers pilot.
- **QuestPDF licensing** — verify free-tier (Community License < $1M revenue). Confirm with team lead.
- **PDF size** — 12 months of invoices can exceed 10 MB; compress images, PDF/A-2b conformance for long-term archival.
