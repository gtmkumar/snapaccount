The following controls were verified in Phase 5 (2026-04-05) and should not be re-flagged unless the relevant files change:

- **CORS:** All 11 services use `WithOrigins(AdminPanel, Mobile)` from config with fallback to prod URLs. Verified in AuthService, GstService, SubscriptionService.
- **Hangfire:** `HangfireRoleAuthorizationFilter("SYSTEM_ADMIN")` in AuthService Program.cs.
- **OTP entropy:** `RandomNumberGenerator.GetInt32(100000, 1000000)` in OtpService.cs.
- **Root .gitignore:** Present at repo root; covers .env, credentials, certs, bin/, appsettings.*.json.
- **Audit log immutability:** PostgreSQL rules `no_delete_audit_log` and `no_update_audit_log` in `database/shared/V2__audit_log_immutability.sql`.
- **Rate limiting:** AuthService uses sliding window (5/10min) on OTP endpoints; stub services use fixed window (100/min).
- **RBAC pipeline:** `PermissionBehavior<TRequest,TResponse>` registered in MediatR pipeline; `[RequiresPermission]` attribute available.
- **PAN encryption:** AES-256-CBC in `AesPanEncryptionService.cs` (CBC — upgrade to GCM is NEW-003, LOW).
- **Certificate pinning:** Infrastructure in place via `react-native-ssl-pinning`; placeholder hashes need replacement before prod (INFO-001).
- **Screenshot prevention:** `useSensitiveScreen()` on 8 screens (ITR, GST, Loan, Report).
- **SecureStore partialize:** `panNumber: undefined` excluded from user, currentOrganization, and organizations[].
- **document-service-sa:** `objectCreator + objectViewer` (not objectAdmin) in setup.sh.
- **HSTS:** `Strict-Transport-Security` header with 1-year max-age, includeSubDomains, preload in nginx.conf.
- **Signed URL:** `GoogleCredential.GetApplicationDefaultAsync()` + `UrlSigner.FromCredential()` — no service account file path.
- **Audit log partitioning:** `shared.create_audit_log_partitions(12)` function + immediate call on migration.
- **TaxComputation integrity:** SHA-256 hash of canonical JSON inputs; `VerifyIntegrity()` uses `CryptographicOperations.FixedTimeEquals`.
- **RLS:** Confirmed present on user-owned tables (established in Phase 4 audit, not re-examined in Phase 5 as database schema unchanged).
- **Data localization:** All GCP resources in asia-south1 (Mumbai).
- **Firebase token storage:** Expo SecureStore (iOS Keychain / Android Keystore).
