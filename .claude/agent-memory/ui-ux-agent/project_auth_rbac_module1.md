Auth & RBAC vertical slice (Module 1, started 2026-05-29). Specs in docs/design/screens/web-admin/auth-rbac-*.md; components appended to component-library.md under "Auth & RBAC Module (Module 1)". No tokens added — fully reused tokens.json v2.0.0.

**Why:** Multi-tenant B2B with custom (data-driven) roles + constrained delegation. Scope: .claude/orchestrator/auth-rbac-module-scope.md (single source of truth).

**How to apply:**
- Pattern = Dribbble HRIS "Permissions settings": left role list (search + create), right permissions grouped by module into collapsible sections, row toggles, "select all in module", sticky dirty save bar.
- CRITICAL delegation rule (cross-cutting): a perm/role control is interactive ONLY if its key ∈ caller's grantable set (`GET /auth/me/grantable-permissions`). Else disabled/greyed: `--text-tertiary` text + 14px lucide Lock + disabled Switch (neutral.200/700 track, neutral.400 thumb, aria-disabled, out of tab order, lock icon stays focusable) + Tooltip. Existing grant values NEVER hidden. UI is assistance only — server authoritative; 403 escalation → toast + revert rejected rows (rose error.500 left-border).
- System roles read-only for ORG_ADMIN (banner + Duplicate). SUPER_ADMIN edits global/system roles.
- New components (all extend existing primitives): RoleListItem, PermissionModuleSection, PermissionRow, DirtySaveBar, CreateRoleDialog, OrgSwitcher (SUPER_ADMIN), CreateOrgDialog, InviteMemberDialog (extends TeamPage InviteDialog), SegmentedControl, PhoneField (+91 chip + 10-digit), RoleCard (radio w/ disabled delegation variant), PasswordStrengthMeter.
- Members page re-scopes existing TeamPage pattern to org endpoints (teamApi.ts stubs → /auth/org/*). Self-guard + last-admin guard on member actions.
- Invite acceptance = PUBLIC token route /invite/:token, reuses auth.md card shell; GET validates → set-password (new) OR link-account (Firebase Google/Apple/password); terminal cards for expired/revoked/accepted/invalid. DPDP terms consent required.
