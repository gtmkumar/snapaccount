Phase 6C delivered 6 mobile screens (LoanHub, Eligibility, Consent, Application, PackagePreview, Status) + 4 admin pages (LoansList, LoanDetail, BankCommunications, PartnerBanksSettings) + component-library addendum.

**Why:** Loan Hub is the flagship P0 differentiator — packaged loan applications with watermarked PDF + DPDP-compliant consent flow. Touches every prior phase's data (6A accounting, 6B GST, 6D ITR docs).

**How to apply (future phases):**
- Reuse `CelebrationOverlay` primitive for any future success moment (Phase 6F will share it). API: `kind`, `headline`, `subline`, optional confetti.
- Reuse the **scroll-to-bottom-before-enable** consent pattern for any future legal acceptance (DPDP). Pair with biometric re-auth on submit.
- Loan disclaimer copy is canonical and required on 4 surfaces: preview screen, PDF footer (server), email body, admin detail page. Do NOT alter without legal sign-off.
- Watermark text is canonical: `"Generated by SnapAccount | {orgName} | {date} | Package ID: {id} | Not a CA certification"`.
- Status enum (DRAFT/SUBMITTED/UNDER_REVIEW/DOCS_REQUESTED/APPROVED/REJECTED/DISBURSED/CLOSED) → Badge variants documented in component-library §"Phase 6C — Loan Hub Additions".
- Adapter type chips: email/rest/oauth — reused anywhere we have pluggable external integrations.
- Consent versioning: `consent_text_version` field; UI must surface version + date; never re-display "current" body for prior signatures.
- Biometric re-auth invoked at 3 surfaces: view-package, submit-package, sign-each-consent. View and submit are separate gates.

**New component primitives added (for frontend-dev + mobile-dev pre-staging):**
LoanProductCard, BadgeQual, EligibilityHintRow, ConsentSignatureBlock, ScrollHintBanner, PdfViewerMobile, PackageMetaStrip, DisclaimerCard, BankAdapterTypeBadge, BankCommStatusBadge, BankHealthBadge, ConsentAuditCard, PayloadViewer, ProductChipsEditor, LogoUploader, ETACountdownCard, CelebrationOverlay.

**Open recommendation logged in report-back:** Show indicative interest rate ranges per bank in catalog with disclaimer "indicative only" — yes; rationale: SME users compare rate ranges first when shopping for a loan; without it they bounce to bank websites.
