## Key design decisions

- **Approve → accounting event**: publishes `OcrCompletedAccountingPayload` (not `OcrCompletedEvent`) directly to `snapaccount.document.ocr.completed` via `IDocumentEventPublisher`. The shape matches `AccountingService.OcrResultSubscriber.OcrCompletedPayload` exactly. Idempotent (already-APPROVED returns success, no re-emit).
- **Valid approve states**: `OCR_COMPLETE` and `IN_REVIEW` only. Handler returns `ErrorType.Validation` for invalid transitions (not throws — result pattern).
- **ArchiveDocument**: handler already existed (`ArchiveDocumentCommand.cs`) — only the endpoint route was missing. Just wired it.
- **Clarification notification**: `DOC_CLARIFICATION_REQUESTED` is NOT in the 26-event catalog. Left a TODO comment in handler pointing to the proposed event code. Do NOT add to NotificationEventCatalog without orchestrator approval.
- **Permission name**: `document.review` (new — consistent with existing `document.archive`).
- **Mock DbSet in tests**: custom `InMemoryAsyncQueryProvider` wraps `IQueryProvider` using dynamic dispatch for `ExecuteAsync<TResult>` to avoid `AmbiguousMatchException` on .NET 10.
