Phase 6F security hotfix completed 2026-04-25.

**Test baseline going in**: 235 tests (1 pre-existing failure)
**Test baseline coming out**: 325 tests, 1 pre-existing failure (LoanPackagePreviewScreen watermark `children` is array not string)

**Why:** security-reviewer flagged 4 findings; SEC-048/050 were 3rd-phase deferrals — mandatory ship.

**How to apply:** Reference for Phase 7. All biometric screens follow the same async LocalAuthentication pattern. Consent catalog follows TanStack Query with graceful 404 fallback.

## SEC-054 (FIXED)
- Bug: ChatDetailScreen.tsx line 232: `buildChatHubConnection(HUB_BASE_URL, async () => null)` — null JWT
- Fix: `() => FirebaseAuth.getIdToken()` (imported from `../../lib/firebase`)
- Pattern: Always pass a real async getter to buildChatHubConnection, never a null factory

## SEC-055 + SEC-034 (FIXED)
- Added `isValidUuid(s: string): boolean` exported from notificationRouter.ts
- Regex: `/^[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}$/i`
- Applied to ALL 6 id-param routes: callback(id), document(id), chat_message_received(threadId), loan_disbursed(loanId), loan_approved(loanId)
- On invalid UUID: fall through to default (no navigation)

## SEC-048 (FIXED)
- Installed expo-local-authentication ~15.0.2 via `npm install --legacy-peer-deps` (pre-existing firebase peer dep conflict)
- Jest mock: `src/__mocks__/expoLocalAuthentication.ts` — added to package.json moduleNameMapper
- Pattern in all 3 screens:
  1. `await LocalAuthentication.hasHardwareAsync()` — if false, fall back to Alert.alert
  2. `await LocalAuthentication.authenticateAsync({ promptMessage: t('mobile.biometric.confirm'), fallbackLabel: t('common.usePin'), disableDeviceFallback: false })`
  3. if `result.success` → proceed; else abort silently (user can retry)
- Screens updated: LoanConsentScreen (handleSign), LoanPackagePreviewScreen (useEffect view-gate + handleSubmitConfirm), UserApprovalScreen (handleBiometric)
- Key: handleBiometric/handleSign must be `async` and called with `await act(async () => {})` in tests

## SEC-050 (FIXED — mobile side)
- Added `getConsentCatalog()` to loans.ts → GET /loans/consents/catalog
- Types: `ConsentCatalogEntry { consentType, textVersion, effectiveDate }`, `ConsentCatalogResponse { items }`
- LoanConsentScreen: `useQuery(['loan-consent-catalog'], getConsentCatalog, { retry: false, staleTime: 5min })`
- `getConsentVersion(consentType)` resolves version from catalog or falls back to `FALLBACK_CONSENT_VERSION = '1.4'`
- `CONSENT_VERSION = '1.4'` constant renamed to `FALLBACK_CONSENT_VERSION` to clarify it's a fallback only
- Consent catalog mock in LoanConsentScreen.test.tsx returns `getConsentCatalog` from loans mock

## P6-HANDOFF-25 (BACKEND)
- backend-agent must implement: `GET /loans/consents/catalog`
- Response shape: `{ items: [{ consentType: 'CREDIT_BUREAU'|'DATA_SHARE_WITH_BANK'|'DISBURSEMENT_MANDATE', textVersion: string, effectiveDate: string }] }`
- Purpose: DPDP audit trail — consent record must reference exact text version shown to user
- Until endpoint ships: mobile falls back to '1.4' (gracefully — `retry: false`, `staleTime: 5min`)
